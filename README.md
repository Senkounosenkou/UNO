# kintone UNO Game カスタマイズ

kintone 上で動作するマルチプレイヤーUNOゲームのカスタマイズ用リポジトリです。
このリポジトリは、**Lobbyアプリ（待合室）**と**Boardアプリ（ゲーム盤面）**の2つのJavaScriptカスタマイズファイルを、共通のモデル・ビューからバンドル（ビルド）して出力します。

---

## 📂 ディレクトリ構成

```
UNO/
├── dist/                 # ビルド成果物（kintoneにアップロードするファイル）
│   ├── lobby_action.js   # ロビーアプリ用のバンドルファイル
│   └── UNO.js            # ゲーム盤面アプリ用のバンドルファイル
├── src/                  # ソースコードディレクトリ
│   ├── constants.js      # ゲームで使う共通定数 (色定義など)
│   ├── model.js          # 純粋なビジネスロジック (デッキ生成、シャッフル、ルール判定など)
│   ├── model.test.js     # model.js に対する単体テスト (Vitest)
│   ├── view.js           # UI表示、HTML構築、CSSスタイルの注入
│   ├── lobby_main.js     # ロビーアプリ用のエントリーポイント
│   └── main.js           # ゲーム盤面アプリ用のエントリーポイント
├── build.js              # 2つのJavaScriptを個別にIIFE形式でビルドするためのスクリプト
├── vite.config.js        # Vitestテスト用の構成ファイル
├── package.json          # 依存関係およびスクリプト定義
└── README.md             # 本書
```

---

## 🛠️ 環境構築・開発手順

### 1. 依存関係のインストール
プロジェクトのルートディレクトリで以下を実行して、必要なパッケージ（Vite, Vitest）をインストールします。
```bash
npm install
```

### 2. 単体テストの実行
ゲームルール（カードの提出可否など）の検証テストを実行します。
```bash
npm run test
```

### 3. ビルドの実行
ソースファイルをバンドルし、kintoneが直接読み込める形式（IIFE形式）にコンパイルして `dist/` ディレクトリに出力します。
```bash
npm run build
```
実行後、`dist/lobby_action.js` と `dist/UNO.js` が自動で生成または更新されます。

---

## 🚀 kintone への適用手順

ビルドによって生成されたファイルを、kintone の各アプリ設定画面からアップロードして保存します。

### ① ロビー（待合室）アプリへの設定
1. ロビーアプリの **「設定 ＞ JavaScript/CSSでカスタマイズ」** を開きます。
2. PC用のJavaScriptファイルに、以下をアップロードします。
   * `dist/lobby_action.js`
3. アプリを更新します。

### ② ゲーム盤面（Board）アプリへの設定
1. ゲーム盤面アプリの **「設定 ＞ JavaScript/CSSでカスタマイズ」** を開きます。
2. PC用のJavaScriptファイルに、以下をアップロードします。
   * `dist/UNO.js`
3. アプリを更新します。

---

## 📝 チーム開発時の注意点

* **ソースコードの変更**:
  * 開発時は直接 `dist/` 内のファイルを編集せず、必ず `src/` 配下のファイルを修正してください。修正後は必ず `npm run build` を実行して成果物を生成し、テストが通ることを確認してからコミットしてください。
* **ビジネスロジックの追加**:
  * ルール追加や変更の際は、[src/model.js](src/model.js) を変更すると同時に、必要に応じて [src/model.test.js](src/model.test.js) にもテストを追加してください。
